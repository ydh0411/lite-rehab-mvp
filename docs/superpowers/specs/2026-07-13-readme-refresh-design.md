# LiteRehab README 轻量美化设计

## 目标

在不增加截图、宣传图或额外网页的前提下，重新整理 GitHub 首页，使其准确反映当前已经跑通的 LiteRehab 系统，并让读者能快速理解系统组成、当前状态和启动方式。

## 修改范围

- 同步更新英文首页 `README.md` 和中文入口 `README_zh.md`。
- 不修改 ESP-IDF 固件、Python Dashboard、MaixCAM2 程序或训练模型。
- 不加入未经验证的性能、医疗效果或硬件能力声明。

## 页面结构

1. 顶部使用居中的项目标题、简短副标题和语言入口。
2. 添加少量静态技术徽章，展示 ESP-IDF、ESP32-S3、Python、MaixCAM2 RTSP 和 CNN-BiGRU；不依赖需要账户授权的动态服务。
3. 使用简短状态表列出已经实现并验证的功能：双板 BLE、独立 RTSP 摄像头、MediaPipe 姿态、CNN-BiGRU 自动加载、CSV 同步记录和声光反馈。
4. 使用 GitHub 原生 Mermaid 流程图表达两条输入链：佩戴端 IMU 经 BLE/串口进入电脑，MaixCAM2 经 RTSP 进入电脑，最后由 Dashboard 融合。
5. 将快速开始调整为当前真实路径：MaixCAM2 默认运行 `MODE = "rtsp"`，通过 USB NCM 地址提供视频；UVC 仅作为可选模式说明。
6. 保留硬件与关键接线摘要，将详细内容链接到现有接线指南。
7. 保留演示动作、模型数据来源、测试命令、项目结构、文档入口和安全声明，但压缩重复解释。

## 必须修正的现状差异

- README 不再将 UVC 描述为当前默认方案。
- 明确独立摄像头只替换视频输入，CNN-BiGRU 和姿态识别仍运行在电脑端。
- 明确 Dashboard 默认加载 `python/models/imu_cnnbigru.pt`。
- 明确模型使用公开小规模上肢 IMU 子集，静止由 ESP32 规则门控，无需用户自行录制训练数据。
- 测试数量更新为 61 项 Python 测试和 3 项 C 主机测试，并说明两套 ESP-IDF 固件构建已纳入完整检查。

## 风格约束

- 页面保持工程项目风格，不使用夸张营销语言。
- 标题层级不超过三级，避免过多折叠区和装饰符号。
- 徽章只用于快速识别技术栈；正文仍提供可访问的文字说明。
- 英文和中文页面使用相同的信息结构，命令与文件路径保持一致。

## 验收标准

- 两份 README 中不存在“UVC 是当前默认模式”或“模型尚未提供”等过时描述。
- Mermaid 语法、Markdown 链接和代码块结构可被 GitHub 正常解析。
- README 中的启动命令与 `scripts/start_maixcam2_demo.sh` 当前接口一致。
- `rg` 检查不再发现旧测试数量 `49` 或旧模型文件名。
- 完整项目测试保持通过；本次只修改文档，因此不改变运行逻辑。
